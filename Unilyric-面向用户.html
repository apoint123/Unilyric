🌟 主要特性

UniLyric 提供了丰富的功能，满足您处理歌词的各种需求

多平台支持

从 QQ音乐、网易云音乐、酷狗音乐和 AMLL TTML 数据库获取歌词，一站式满足您的需求。

格式转换

支持 LRC、QRC、KRC、YRC、TTML、Lyricify 等多种歌词格式的相互转换。

智能搜索

基于歌曲元数据（标题、艺术家、专辑）的智能匹配算法，快速找到最匹配的歌词。

批量处理

支持批量转换和合并多个歌词文件，提高处理效率，节省您的时间。

多语言支持

自动处理中文简繁转换，支持翻译和罗马音歌词，满足多语言需求。

图形界面

基于 egui 的现代化跨平台 GUI 界面，简洁直观，易于操作。

对于普通用户

下载预编译版本

前往 Releases 页面 下载最新的预编译版本：

  • Windows: 下载 .exe 可执行文件
  • macOS: 下载 .dmg 安装包或 .app 文件

基本使用

  1. 启动 UniLyric 应用程序
  2. 在搜索栏输入歌曲名称和艺术家
  3. 选择喜欢的歌词版本
  4. 选择目标格式进行转换
  5. 保存或导出歌词文件

对于开发者

环境要求

  • Rust: 1.70+ (推荐使用 nightly 工具链以获得最佳性能)
  • Cargo: 最新版本
  • 系统: Windows, macOS, 或 Linux

从源码构建


# 克隆仓库
git clone https://github.com/apoint123/Unilyric.git
cd Unilyric

# 构建整个工作区
cargo build --release --workspace

# 构建特定包
cargo build --release -p lyrics_helper_rs     # 主库包
cargo build --release -p Unilyric             # GUI应用
cargo build --release -p lyrics_helper_core   # 核心数据结构

# 运行图形界面应用程序
cargo run --release -p Unilyric

# 运行命令行工具
cargo run --release -p lyrics_helper_rs -- --help

# 为特定平台构建
cargo build --release --target x86_64-pc-windows-msvc

安装依赖


# 更新 Rust 工具链
rustup update

# 安装 nightly 工具链（可选，用于性能优化）
rustup toolchain install nightly
rustup default nightly

📖 详细使用指南

了解如何充分利用 UniLyric 的各项功能

图形界面操作

  1. 1
    搜索歌曲

    在主界面的搜索框中输入歌曲信息，可以包括歌曲名、艺术家和专辑

  2. 2
    选择提供商

    从下拉菜单选择音乐平台（QQ、网易云、酷狗等），可以选择多个平台同时搜索

  3. 3
    预览歌词

    查看搜索结果并选择最匹配的版本，点击可预览完整歌词内容

  4. 4
    格式转换

    选择目标输出格式，可以设置额外的转换选项如编码、时间偏移等

  5. 5
    导出保存

    将转换后的歌词保存到本地文件，可以选择保存路径和文件名格式

命令行使用

UniLyric 也提供命令行接口，适合批量处理和自动化：

搜索歌曲


lyrics_helper_rs search "歌曲名" --artist "艺术家"

转换歌词格式


lyrics_helper_rs convert input.lrc --output output.ttml --format ttml

批量处理文件夹


lyrics_helper_rs batch-convert ./input_folder/ --output-dir ./output/ --format krc

配置文件

创建 config.toml 文件来自定义设置：


[providers]
qq_music = true
netease = true
kugou = true
amll_ttml_database = true

[conversion]
default_format = "ttml"
auto_convert_chinese = true

[ui]
theme = "dark"
language = "zh-CN"

配置文件位置：

  • Windows: %APPDATA%\UniLyric\config.toml
  • macOS: ~/Library/Application Support/UniLyric/config.toml
  • Linux: ~/.config/unilyric/config.toml
