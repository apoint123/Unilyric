项目概览

UniLyric 是一个专业的歌词转换器，支持从多个音乐平台获取歌词并进行格式转换。项目采用 Rust 语言开发，使用 Cargo 工作区管理多个 crate。

项目特点

  • 多平台歌词获取
  • 多种格式转换支持
  • 高效的 Rust 实现
  • 直观的图形界面

代码架构

项目采用分层架构，包含以下主要组件：

核心 crate

lyrics_helper_core

数据结构和基础类型定义，被其他 crate 依赖

lyrics_helper_rs

主要业务逻辑，包含歌词提供者、搜索、转换功能

ttml_processor

专门处理 Apple Music TTML 格式的库

Unilyric

图形界面应用程序，基于 eframe/egui

核心架构特性

异步处理

使用 Tokio runtime 处理网络请求，支持高并发搜索

模块化提供商

每个音乐平台作为独立的 Provider 实现，易于扩展

格式抽象

统一的歌词格式转换系统，支持多输入多输出格式

智能匹配

基于歌曲元数据的智能搜索算法，支持多种匹配策略

歌词提供者支持

支持的音乐平台：

QQ音乐 网易云音乐 酷狗音乐 AMLL TTML 数据库

完整功能支持：

  • 搜索功能 - 歌曲、专辑、艺术家搜索
  • 歌词获取 - 多语言歌词、翻译、罗马音
  • 专辑信息 - 元数据、曲目列表、封面
  • 登录支持 - 二维码、Cookie登录验证
  • 会话管理 - 状态保存和恢复

提供商特性对比：

QQ音乐、网易云音乐支持完整功能；酷狗音乐支持搜索和歌词；AMLL数据库专门处理TTML格式

格式转换支持

支持多种歌词格式的输入和输出：

LRC
QRC
KRC
YRC
TTML
Lyricify

高级转换特性：

  • 多文件合并 - 主歌词、翻译、罗马音文件合并
  • 时间轴同步 - 智能时间戳对齐和校正
  • 简繁转换 - 自动中文简繁体转换
  • 元数据保留 - 保持原始文件的元数据信息
  • 批量处理 - 支持文件夹批量转换

开发命令

构建命令 


# 构建整个工作区
cargo build --workspace

# 发布构建（优化性能）
cargo build --release --workspace

# 构建特定包
cargo build -p lyrics_helper_rs     # 主库包
cargo build -p Unilyric             # GUI应用
cargo build -p lyrics_helper_core   # 核心数据结构
cargo build -p ttml_processor       # TTML处理器

# 为特定target构建
cargo build --release --target x86_64-pc-windows-msvc
cargo build --release --target wasm32-unknown-unknown

测试命令 


# 运行所有测试
cargo test --workspace

# 运行特定包的测试
cargo test -p lyrics_helper_rs      # 主库包测试
cargo test -p ttml_processor        # TTML处理器测试
cargo test -p lyrics_helper_core    # 核心数据结构测试

# 运行集成测试（需要网络连接）
cargo test -- --ignored

# 运行特定测试文件
cargo test --test test_name

# 运行特定测试函数
cargo test test_function_name

# 并发运行测试
cargo test --workspace -- --test-threads=4

Lint 检查 


# Clippy 检查全部代码
cargo clippy --workspace -- -D warnings

# 单个包的 Clippy 检查
cargo clippy -p lyrics_helper_rs -- -D warnings
cargo clippy -p Unilyric -- -D warnings
cargo clippy -p ttml_processor -- -D warnings

# 格式化所有代码
cargo fmt --all --check

其他命令 


# 生成依赖树和查看依赖
cargo tree --workspace
cargo tree -p lyrics_helper_rs
cargo outdated

# 更新依赖
cargo update

# 文档生成
cargo doc --workspace --no-deps
cargo doc --workspace --open

# 清理构建缓存
cargo clean

CI/CD 配置

GitHub Actions 配置位于 Unilyric/.github/workflows/release.yml

  • 在 main 分支推送时自动构建发布版本
  • 使用 nightly Rust 工具链
  • 生成 Windows 可执行文件并创建发布

2

3

性能优化

4

多语言支持

主要支持中文歌曲和相关功能

开发要点

  • 主要入口点：lyrics_helper_rs/src/lib.rs:276 和 Unilyric/src/main.rs
  • 异步架构：使用 Tokio runtime 处理网络请求，支持高并发搜索
  • 设计模式：策略模式、工厂模式、适配器模式、观察者模式
  • 错误处理：统一的错误类型 LyricsHelperError，包含详细错误上下文
  • 日志调试：集成 tracing 库，设置日志级别：RUST_LOG=lyrics_helper_rs=debug
  • WASM 支持：支持编译为 WebAssembly，可用于 Web 应用
  • 会话管理：持久化登录状态，支持会话导入导出
  • 取消支持：使用 CancellationToken 支持操作取消
